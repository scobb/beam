## Codebase Patterns
- beam.js is served dynamically via a Cloudflare Worker (tracking.ts BEAM_JS constant) — NOT from a static file. The beam-js/beam.js file is the unminified source kept in sync.
- Production deploys use `npx wrangler deploy -c wrangler.deploy.toml` (not wrangler.toml) to avoid the zone-level route auth error. beam-privacy.com is a Custom Domain (not a route), so no zone permission needed.
- Staging deploy route registration fails (auth error on CF zones API) — pre-existing issue. Smoke tests should be run directly against beam-privacy.com.
- sendBeacon with Blob(type:'application/json') sends credentials:include by default, which breaks CORS when the server responds with Access-Control-Allow-Origin: *. Use fetch(keepalive:true, credentials:'omit') instead.

## 2026-04-13 - BEAM-211
- Investigated "Waiting for data" on cards.keylightdigital.dev Beam dashboard
- Root cause: `navigator.sendBeacon` sends with `credentials: 'include'` by default. When beam.js is loaded cross-origin (script src = beam-privacy.com, page = cards.keylightdigital.dev), the CORS preflight fails because the server returns `Access-Control-Allow-Origin: *` which is not allowed when credentials mode is 'include'. The beacon was silently dropped — no pageviews were recorded.
- Fix: replaced `sendBeacon` with `fetch(keepalive:true, credentials:'omit')` in both `src/routes/tracking.ts` (production BEAM_JS) and `beam-js/beam.js` (unminified source). `credentials:'omit'` means no cookies sent, so wildcard CORS works fine.
- Files changed: `src/routes/tracking.ts`, `beam-js/beam.js`, `test/smoke/smoke.spec.ts`
- Deployed to prod via `npx wrangler deploy -c wrangler.deploy.toml`
- Smoke tests: 6 tracking/CORS tests passed against production
- **Learnings for future iterations:**
  - sendBeacon is NOT safe for cross-origin analytics — always use fetch+keepalive+credentials:omit
  - The Playwright browser session caches beam.js — CORS errors in the persistent session after deployment are from the old cached script, not the live version. Smoke tests in fresh contexts are the reliable check.
  - Staging route registration is broken (CF auth) — use prod smoke tests
